Verifying the Source Code

The Alarm service is comprised of one normal contract and five library contracts. Verification of the alarm source requires independent verification of each of these contracts.

  • Alarm: 0x07307d0b136a79bac718f43388aed706389c4588
  • ScheduledCallLib: 0x5c3623dcef2d5168dbe3e8cc538788cd8912d898
  • ResourcePoolLib: 0xd6bbd16eaa6ea3f71a458bffc64c0ca24fc8c58e
  • AccountingLib: 0x7de615d8a51746a9f10f72a593fb5b3718dc3d52
  • GroveLib: 0x920c890a90db8fba7604864b0cf38ee667331323
  • StringLib: 0x443b53559d337277373171280ec57029718203fb

While the source code for all libraries and contracts can be found on this page, due to a compiler bug with solc, you cannot verify the source using the merged source found below. Instead you will need to use the published sources which are available in the 0.4.0 release on github

To verify the compiled code of the contract, you will need to compile the source below with the solc compiler. You will likely need to use the same version used when deploying the contract,

$ solc --version
solc, the solidity compiler commandline interface
Version: 0.1.6-d41f8b7c/Release-Darwin/clang/JIT linked to libethereum-1.1.0-83bfca60/Release-Darwin/clang/JIT

The source was compiled with the command solc --optimize --bin --bin-runtime contracts/Alarm.sol libraries/* with the command being run from the root of the github project. This will produce unlinked source code, which means that there will be placeholders for the addresses of the library contracts. You can either compile with linking enabled or manually fill in the contract addresses above to get the compiled binary.

$ solc --bin --bin-runtime --optimize contracts/Alarm.sol libraries/* | solce --link --libraries AccountingLib:0x7de615d8a51746a9f10f72a593fb5b3718dc3d52,GroveLib:0x920c890a90db8fba7604864b0cf38ee667331323,ScheduledCallLib:0x5c3623dcef2d5168dbe3e8cc538788cd8912d898,StringLib:0x443b53559d337277373171280ec57029718203fb,ResourcePoolLib:0xd6bbd16eaa6ea3f71a458bffc64c0ca24fc8c58e
