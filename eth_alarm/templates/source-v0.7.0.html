Verifying the Source Code

The Alarm service is comprised of one normal contract and four library contracts. Verification of the alarm source requires independent verification of each of these contracts.

  • Scheduler: 0xe54d323f9ef17c1f0dede47ecc86a9718fe5ea34
  • SchedulerLib: 0x7c1eb207c07e7ab13cf245585bd03d0fa478d034
  • CallLib: 0x1deeda36e15ec9e80f3d7414d67a4803ae45fc80
  • AccountingLib: 0x89efe605e9ecbe22849cd85d5449cc946c26f8f3
  • GroveLib: 0x6c8f2a135f6ed072de4503bd7c4999a1a17f824b

The source code for this release can be found 0.7.0 release on github

To verify the compiled code of the contract, you will need to compile the source below with the solc compiler. You will likely need to use the same version used when deploying the contract,

$ solc --version
solc, the solidity compiler commandline interface
Version: 0.2.0-d2f18c73/Release-Darwin/unknown/int linked to libethereum-1.1.1-377f1a37/Release-Darwin/unknown/int

The source was compiled with the command solc --optimize --bin --bin-runtime contracts/* libraries/* with the command being run from the root of the github project. This will produce unlinked source code, which means that there will be placeholders for the addresses of the library contracts. You can either compile with linking enabled or manually fill in the contract addresses above to get the compiled binary.

$ solc --optimize --bin --bin-runtime contracts/* libraries/* | solc --libraries SchedulerLib:0xe54d323f9ef17c1f0dede47ecc86a9718fe5ea34,GroveLib:0x7c1eb207c07e7ab13cf245585bd03d0fa478d034,CallLib:0x1deeda36e15ec9e80f3d7414d67a4803ae45fc80,AccountingLib:0x89efe605e9ecbe22849cd85d5449cc946c26f8f3 --link
